# Ideation Memory Schema

Use this schema when `direction-evolution` runs in `memory-update` mode.

## Recommended Sections

```text
# Ideation Memory

## Promising Directions
## Failed or Deprioritized Directions
## Open Questions to Revisit
## Decision Heuristics
```

## Entry Shape

- `Direction`: short name
- `Core problem`: what gap or tension it addresses
- `Why it looked promising`: strongest positive evidence
- `Why it stalled or failed`: strongest negative evidence
- `Next status`: pursue, revisit later, merge into another direction, or avoid

## Good Failure Labels

- `Novelty overlap`
- `Resource mismatch`
- `Weak effect`
- `Confounded setup`
- `Problem misframed`
- `Evaluation too narrow`
- `Needs different benchmark`
