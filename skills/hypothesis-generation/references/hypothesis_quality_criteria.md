# Hypothesis Quality Criteria

Use these criteria to compare competing hypotheses.

## 1. Testability

Can the hypothesis be examined empirically?

Questions:

- Can we design a study or experiment around it?
- Are the outcomes measurable?
- Are there multiple independent ways to test it?

## 2. Falsifiability

What observation would show the hypothesis is wrong?

Warning signs:

- built-in escape clauses
- vague modifiers like "may depend on context" without specifying which context
- post-hoc reinterpretations that fit any result

## 3. Parsimony

Does the hypothesis invoke only the mechanisms it needs?

Notes:

- simple is not always better, but unnecessary speculation is worse
- one well-supported mechanism is usually stronger than many weakly supported ones

## 4. Explanatory Power

How much of the observed phenomenon does it actually explain?

Strong hypotheses:

- account for multiple observations
- explain anomalies, not just the easy cases
- generate new expectations instead of merely fitting old facts

## 5. Scope

How broadly does the hypothesis apply?

Questions:

- only this case, or a broader class of cases?
- are the boundaries explicit?
- could it generalize across systems, settings, or scales?

## 6. Consistency with Established Knowledge

Does it align with well-supported principles, or if not, does it have unusually strong justification?

## 7. Insight / Novelty

Does it offer new understanding, a new mechanism, or a useful reframing?

## Comparative Use

No hypothesis needs to dominate on every axis.

Use the criteria to ask:

- which hypothesis is most testable right now?
- which is most explanatory?
- which is simplest while still plausible?
- which experiment would most efficiently separate them?
