# Ideation Memory Schema

Use this reference to keep `memory/ideation-memory.md` compact, reusable, and decision-oriented.

## Recommended Sections

```text
# Ideation Memory

## Promising Directions
## Failed or Deprioritized Directions
## Open Questions to Revisit
## Decision Heuristics
```

## Entry Shape

Each direction entry should capture:

- `Direction`: short name
- `Core problem`: what gap or tension it addresses
- `Why it looked promising`: strongest positive evidence
- `Why it stalled or failed`: strongest negative evidence
- `Next status`: pursue, revisit later, merge into another direction, or avoid

## Good Failure Labels

- `Novelty overlap`
- `Resource mismatch`
- `Weak effect`
- `Confounded setup`
- `Problem misframed`
- `Evaluation too narrow`
- `Needs different benchmark`

## Decision Heuristics

This section should store reusable lessons, for example:

- Prefer directions that expose hidden failure modes over ones that only add parameters
- Reject ideas that depend on infrastructure unavailable in the current template
- Do not trust metric gains without a stronger baseline or slice analysis

## When To Update

- After a shortlist is finalized
- After a direction is rejected for novelty or feasibility
- After an experiment invalidates a core assumption
- After a pivot reveals a better framing of the same underlying problem
